# 專案提案書 (Project Proposal)

## 1. 專案名稱
逢甲大學資訊工程學系 (IECS) 教授專業查詢系統

## 2. 專案目標
建立一個直覺、易用的網頁應用程式，專門針對「逢甲大學資工系」的師資陣容進行統整。讓學生能夠快速查詢資工系教授的專業領域與聯絡資訊，解決學生在選課或尋找專題指導教授時資訊零散的問題。

## 3. 系統架構規劃
根據需求，本專案將採用前後端分離架構，並限制使用純 HTML, CSS, JavaScript (不使用 TypeScript 與外部 UI 框架如 Tailwind)。
*   **前端 (Client)**: `/client` - 使用 Vue.js + Vite 建構，負責使用者介面呈現與互動。
*   **後端 (Server)**: `/server` - 使用 Node.js + Express 建構，純 API 伺服器 (API only)。
*   **資料庫**: SQLite - 輕量級關聯式資料庫，便於開發與部署。

## 4. 參考與介面設計概念
*   **參考專案**: 依據指定的 GitHub Repo (`RateMinistere`) 進行架構與介面的參考。
*   **介面樣式**: 參考該 Repo 與提供的截圖，首頁將包含：
    *   **快速篩選按鈕**: 對應資工系官網的分類（例如：系主任、專任教師、兼任教師、講座教授等）。
    *   **整合搜尋列**: 允許使用者輸入「教授姓名」或「專業領域關鍵字」進行模糊搜尋。

## 5. 資料來源與擷取策略
*   **目標網址**: 逢甲大學資訊工程學系師資網頁 (`https://www.iecs.fcu.edu.tw/teacher/#FT`)。
*   **爬蟲策略**: 為了避免被校方伺服器封鎖 (Ban)，我們將撰寫 Node.js 爬蟲腳本 (`scraper.js`)，於後台**一次性**抓取網頁原始碼並解析教授資料（姓名、職稱、研究專長、信箱），清洗後全數存入本機的 SQLite 資料庫中。
*   **前端查詢**: 使用者在網頁上的所有查詢動作，皆只會對我們自己的 SQLite 資料庫進行檢索，完全不會對學校網站發送即時請求。
